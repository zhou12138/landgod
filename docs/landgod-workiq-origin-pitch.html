03 · 可信落地：Agent 如何调用本地能力

HTTP 进 Gateway，WebSocket 到 Worker，本地工具执行。

Worker 通过 WebSocket 发布能力；Agent 通过 HTTP 调 Gateway；Gateway 路由签名 tool_call。
真实工作流
Register · Publish · Call

A. 发现入口 + 发布能力

Agent 找到 Gateway；Worker 发布受控能力目录。

Skill/config 发现 Gateway；toolRegistry 包装内置工具 + MCP 工具；update_tools 发布能力。

B. 路由 + 执行工具调用

HTTP 请求进入，Gateway 路由并签名，Worker 本地执行。

入口: Skill/config 提供 Gateway + /tool_call。
路由: 按 connectionId、clientName 或 labels 选择 Worker。
执行: 先校验，再 toolRegistry.callTool。
协议：Agent → HTTP(S) Gateway；Worker → WS(S)；Gateway 将 HTTP /tool_call 转成 WS tool_call。Harness：Gateway governs, Worker executes, Activity proves.
