01 · From WorkIQ Gap to Local Computer Access
LandGod
Problem → Pattern

1. A Concrete Integration Gap

Societas needed to operate WorkIQ for the user, but WorkIQ was only reachable through a local CLI.

Cloud Agent handles reasoning and task planning
WorkIQ capability lived behind workiq CLI
User expects completed actions, not manual relay
The first problem was not “remote control”; it was secure local execution.

2. The Real Boundary

The valuable execution context was local and could not be moved into the cloud.

No public API; local CLI only
AAD tokens, cookies, VPN, device config stay local
No inbound exposure on personal devices
Worker connects outbound; Gateway routes intent
Cloud intelligence needed a governed bridge into the user’s device context.

3. The Pattern Generalizes

WorkIQ exposed a repeatable category: cloud Agents need access to the user’s existing local toolchain.

Generalization: make personal devices governed local execution runtimes for cloud Agents.
Thesis: Local Computer Access is the missing execution layer between cloud Agents and the user’s real work environment.

03 · Trusted Execution: How Agents Call Local Capabilities
Real Workflow
Register · Publish · Execute

A. Register + Publish User-Scoped Tools

Worker connects outbound from the user computer, authenticates, then publishes local capabilities into the MCP Hub catalog.

Registration flow: session_opened → register → buildBindings() → update_tools. Gateway exposes a user-scoped catalog to Agents.

B. Invoke + Forward + Execute

Agent invokes a tool. MCP Hub locates the right computer, forwards a signed call to LandGod, and the Worker executes locally.

Gateway: selects Worker by connectionId, clientName, or labels; signs request.
Worker: validates signature, nonce, body hash, expiry, and tool binding.
Local: invokes toolRegistry.callTool against WorkIQ / CLI / MCP tools.
Protocol: Agent → HTTP(S) Gateway; Worker → WS(S); Gateway converts HTTP /tool_call into WS tool_call. Harness: Gateway governs, Worker executes, Activity proves.
