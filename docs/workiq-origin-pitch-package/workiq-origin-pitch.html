03 · Trusted Execution: How Agents Call Local Capabilities

HTTP to Gateway. WebSocket to Worker. Local tools execute.

Workers publish capabilities over WebSocket. Agents call Gateway over HTTP. Gateway routes signed tool calls.
Real Workflow
Register · Publish · Call

A. Discover + Publish Capabilities

Agent finds Gateway; Worker publishes a governed capability catalog.

Skill/config discovers Gateway. toolRegistry packages built-in + MCP tools. update_tools publishes them.

B. Route + Execute Tool Calls

HTTP request in. Gateway routes and signs. Worker executes locally.

Entry: Skill/config provides Gateway + /tool_call.
Routing: Select Worker by connectionId, clientName, or labels.
Execution: Validate, then toolRegistry.callTool.
Protocol: Agent → HTTP(S) Gateway; Worker → WS(S); Gateway converts HTTP /tool_call into WS tool_call. Harness: Gateway governs, Worker executes, Activity proves.
